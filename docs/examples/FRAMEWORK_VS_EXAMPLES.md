# ตัวอย่างการใช้งาน (Examples) — ปรับให้ตรงกับเวอร์ชันปัจจุบันของ Framework

เอกสารนี้สรุปตัวอย่างที่มาพร้อมกับโปรเจค ชี้ตำแหน่งไฟล์ตัวอย่างที่เกี่ยวข้อง แนะนำการรัน และคำแนะนำในการลบตัวอย่างที่ไม่เกี่ยวข้อง

**สรุปสั้น ๆ:**
- ตัวอย่างที่แถมมากับโปรเจค: `modules/HelloWorld`, `modules/Auth`, ตัวอย่าง Views ใน `app/Views/welcome.php` และตัวอย่าง Controller ใน `app/Controllers/`.
- หากตัวอย่างใดไม่ตรงกับการใช้งานของคุณ สามารถลบได้โดยตรงจากโฟลเดอร์ที่ระบุ (คำแนะนำด้านล่าง).

**ตัวอย่างที่แนะนำให้เก็บไว้:**
- `modules/HelloWorld` : โมดูลตัวอย่างพื้นฐาน แสดงการใช้งานโมดูลและการลงทะเบียนเส้นทาง (route) แบบง่าย
- `modules/Auth` : ตัวอย่างโมดูลระบบยืนยันตัวตน หากคุณใช้งานระบบ authentication ของ framework นี้ ให้เก็บไว้
- `app/Views/welcome.php` : ตัวอย่าง view เริ่มต้น
- `app/Controllers/Web/` และ `app/Controllers/Api/` : ไฟล์ Controller ตัวอย่างที่ใช้แสดงโครงสร้างและแนวทางการเขียน

**ไฟล์และตำแหน่งสำคัญ (paths):**
- โมดูลตัวอย่าง: `modules/HelloWorld/` , `modules/Auth/`
- ตัวอย่าง View: `app/Views/welcome.php`, โฟลเดอร์ `app/Views/_test/` สำหรับตัวอย่างเพิ่มเติม
- ตัวอย่าง Controller: `app/Controllers/Web/`, `app/Controllers/Api/`
- ตัวอย่างการตั้งค่า (ถ้ามี): `config/` (ตรวจสอบไฟล์ที่เกี่ยวข้องก่อนลบ)

**วิธีรันตัวอย่างอย่างรวดเร็ว (development server):**
1. เปิด terminal ไปที่โฟลเดอร์โปรเจค
2. รัน PHP built-in server (ต้องมี PHP ติดตั้ง):

```powershell
php -S localhost:8000 -t public
```

3. เปิดเบราว์เซอร์ที่ `http://localhost:8000` แล้วทดสอบเส้นทางตัวอย่าง เช่น `/hello` หรือเส้นทางที่โมดูลตัวอย่างลงทะเบียนไว้

หมายเหตุ: บางตัวอย่างอาจต้องตั้งค่า `config/app.php` หรือฐานข้อมูลก่อน หากพบข้อผิดพลาดให้ตรวจสอบ `config/database.php` และ `env.php`

**การลบตัวอย่างที่ไม่เกี่ยวข้อง:**
- หากคุณแน่ใจว่าไม่ต้องการตัวอย่างใด ให้ลบทิ้งจาก repository (แนะนำให้ทำ commit แยกเป็นรายการเดียวเพื่อความชัดเจน)
- ตัวอย่างคำสั่ง PowerShell เพื่อลบโฟลเดอร์ตัวอย่าง (เปลี่ยนเป็นโฟลเดอร์ที่ต้องการ):

```powershell
Remove-Item -Recurse -Force .\modules\HelloWorld
Remove-Item -Recurse -Force .\modules\SomeUnusedExample
```

- ก่อนลบ ให้ตรวจสอบว่าไม่มีการอ้างอิงตัวอย่างนั้นใน `routes/web.php`, `routes/api.php`, หรือในไฟล์ `ModuleManager.php` หากมี ให้ลบการอ้างอิงออกก่อน

**แก้ไข/อัปเดตตัวอย่างให้ตรงกับเวอร์ชันของ Framework:**
- ตรวจสอบและปรับ namespace, ชื่อคลาส, หรือ signature ของเมธอดในตัวอย่างให้ตรงกับ `app/Core/` และ `app/Controllers/` ปัจจุบัน
- ตัวอย่างที่ใช้การเรียก library ภายนอก (จาก `vendor/`) ให้ตรวจสอบว่า dependency ยังถูกติดตั้งและเวอร์ชันตรงกัน

**เพิ่มตัวอย่างใหม่:**
- แนะนำให้สร้างโมดูลตัวอย่างใน `modules/YourExampleModule/` โดยมีไฟล์ `YourExampleModule.php` ที่ implements `ModuleInterface` และตัวอย่าง Controller + View ภายใน

**การตรวจสอบหลังปรับปรุง:**
- รันแอป และเปิดหน้าเพจตัวอย่างทั้งหมดที่ยังคงอยู่เพื่อเช็คว่าไม่มี 404 หรือ error
- ตรวจสอบ log ใน `storage/logs/` หากมีข้อผิดพลาดเกิดขึ้น

หากต้องการ ให้ผมช่วย:
- อัปเดตรายละเอียดตัวอย่างให้ตรงกับโค้ดปัจจุบัน (ผมจะสแกนโครงงานและแก้ตัวอย่างให้ทันที)
- หรือลบตัวอย่างที่ไม่ต้องการให้โดยตรง — ระบุรายการที่ต้องการลบได้เลย
